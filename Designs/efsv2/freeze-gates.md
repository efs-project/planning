## A. James ratifications and acknowledgments (the critic's five + two surfaced by the design docs; none blocks reading the design set)

1. **Checkpoint reading** — checkpoints are ordinary reserved-key claims (author-signed state root + through-seq), zero kernel machinery, no head-currency/fork-choice ever. This *interprets* your "no HEAD/CHECKPOINT machinery" ruling; the ops grading tables depend on the interpretation. **Recommend: ratify.**
2. **Kernel state costs** — the enumeration spine (~22–27k gas/record; the from-state-alone reconstruction pledge is unimplementable without it) + the container-scoped discovery index (the consumer apps' #1 demand; without it two of five consumer apps are indexer-dependent for their core read). **Recommend: accept both**; the graded fallbacks are documented but each silently downgrades a pledge. **Now subsumed by decision 9** (the on-chain-completeness bundle), which prices the spine + the full reverse-index surface as one aggregate — decide it there, not here.
3. **KEL timing and scope — SUPERSEDED BY DECISION 10.** The earlier “reserve now, ship by ~2030” framing missed that scoped actors, passkeys, timely revocation, and historical authorization cross the envelope and kernel boundary. Keep the century/PQ urgency, but do not use 2030 as permission to freeze an incompatible v2 surface. See [[kel]].
4. **AppendOnly signer-legibility residual** — a permanent list entry looks like a revocable TAG in the signed bytes (accepted on absence of app evidence; kindTag-alias fallback named). **Recommend: accept; revisit only if clear-signing review finds concrete harm.**
5. **Schedule re-plan** — the Etched artifact as reviewed is ~2,300–2,900 LoC (not the carrier decision's 500–900, which was the narrowest kernel-core reading); the verification budget scales accordingly. **Recommend: ratify the honest number; the +2–4-week carrier estimate likely becomes +4–8 weeks with verification as the driver.**
6. **Safety-class GATE default in the reference SDK** (surfaced by [[read-lens-spec]]) — the shipped default for safety-class gate reads: fail closed vs warn. **Recommend: fail closed; warn only behind an explicit flag.**
7. **String-only scope confirm** (shared [[codex-envelope]]/[[codex-kinds]] open question) — permanent vs v2-scoped. Permanent ⇒ collapse the VAL tail to `(string value)` and burn the datatype-word reservation; v2-scoped ⇒ keep the datatype word as interning-derivation input. **Recommend: v2-scoped as adopted; re-check trigger = a marketplace/sort-range numeric-consumer app.**
8. **Time/ordering fields — envelope-body, FREEZE-SENSITIVE (surfaced 2026-07-08).** Two paired decisions the next Fable pass designs against; both touch the envelope/record body, so they must be settled before the format locks. **(a) Rename `seq` → `order`.** It is a portable per-batch ordering key (which of an author's writes is newer, LWW by `(order, recordDigest)`), *not* a timestamp and *not* a nonce (nonces forbid replay; EFS wants replay). The name should say "ordering," to kill both the "trust it as a clock" and "it prevents replay" footguns. Time-derivation stays as the coordination-free generation trick + the ~now+600s anti-max-out cap — an SDK detail, not the field's meaning. **(b) Add an optional per-record `claimedAt`.** The user's untrusted claim of when they did *that action* — survives batching (one signature over many queued actions shares one `order`, so per-action time would otherwise be lost) and gives timeline/journal apps an honest authoring order. Row-vs-convention-vs-reject like the P2 candidates. Net picture — three honest concepts: `claimedAt` (per action, user claim, untrusted), `order` (per batch, portable ordering, untrusted-as-clock), `admittedAt` (per chain, kernel-stamped, trustworthy — the P1 clock). **Recommend: both; they're cheap now, painful to retrofit after the envelope freezes.**
   - *Adversarial-verification note (2026-07-08, verdict SOUND-WITH-FIXES — [[2026-07-08-fable-fs-prep/verify-time-model]]):* (i) the `seq`→`order` rename is **not cosmetic at the wire** — the field name is in the EIP-712 typeHash string, so it changes the byte-pinned envelope digest, all 42 golden vectors, and the wallet-shown label; must precede the envelope freeze and forces typeHash + golden-suite regeneration. (ii) `claimedAt` is a trailing optional body word like `expiresAt` (no Merkle change) but inherits: fix the canonical order of trailing optional words, `0`=absent encoding, extend the S7 canonical-word check, widen the VAL-tail differential fuzz. (iii) `admittedAt` must be a **stored per-claim state word**, not just an event (for P1 getProof-provability + EIP-4444 log-pruning survival) — real per-record storage, price it into the A2 gas bundle. (iv) ~~One current-doc inconsistency to fix: [[read-lens-spec]] §5.2/§9.C compute freshness age from `tidTime(order)` (the untrusted field) — route freshness through `admittedAt` (trustworthy) with `tidTime` as the labeled fallback.~~ **Superseded 2026-07-10 (FS pass, C1):** `admittedAt` is existence-since evidence ONLY, never a data-freshness anchor (that routing is fail-open even at home); the freshness anchor is the checkpoint **recency beacon** ([[fs-pass-freeze-reservations]] D5). The 'optional' claimedAt wording in (b) is likewise superseded by the C3 always-present shape (uint64, 0 = no testimony) — ratify A.8a/A.8b with these corrections, per [[fs-pass-james-decisions]] §2–3. `claimedAt` opens no protocol attack (it is in no comparator), but extend read-lens P13 with the honesty rule "`claimedAt` is falsifiable against `admittedAt`," anchored on the earliest/home `admittedAt`.

9. **On-chain-completeness bundle — FREEZE-SENSITIVE, one aggregate decision (surfaced 2026-07-10; full analysis [[onchain-completeness]]).** James ruled all core functionality must work on-chain with no dependence on The Graph; the audit found the point-read core is genuinely on-chain but the reverse/enumeration/count layer is on-chain-*durable* yet not on-chain-*queryable* as specced — **five confirmed regressions** from v1 (general backlink, predicate-filtered reverse, address-target backlink, best-of-N mirror ranking, self-enumeration) plus two now-or-never gaps (LIST reverse-membership, REDIRECT cited-by). All ride Etched kernel state that can only be committed **before** the ceremony. Rather than nine separate ticks, this is **one priced bundle** — accept the aggregate gas and every regression closes:
   - **(a) Full-body spine** (not objects-only) — the meta-gate under everything; objects-only makes the entire claim/edge/revocation layer event-only Tier 3. Plus **no-body-elision as an Etched invariant** (so the perf lever can't be taken post-freeze and silently orphan every index). **Must be an explicit signature, never a budget default.**
   - **(b) The reverse index, redesigned** — the `discoverByTarget` postings word must carry **`definitionId`** (the predicate) — without it, predicate-filtered reverse ("which `mirrors`/`act` edges point at X") is O(all-postings-at-X), not O(matches), and stays composability-Tier-3 even after "restoring backlinks." Admit **ADDRESS / LIST / REDIRECT targets + author-keying** in the same word so R1/R3/R5/R6/R8 all resolve on-chain.
   - **(c) A live-revocation story** — reopen B5 (revoke-echo decrement) OR ratify an `isRevokedBatch` reconciliation view **and** rule "counts are advisory, never gate on them" (raw `.length` is attacker-inflatable by spray-then-self-revoke).
   - **(d) Best-mirror view restored** — zero new state (the mirror-TAGs are already kept); a redeployable view ranks them on-chain, restoring the no-infra web3:// pitch.
   **Recommend: accept the full bundle.** On every sub-item the gas-cheapest do-nothing is the Tier-3 outcome, and the aggregate default under budget-silence is an EFS that needs The Graph for backlinks, directory listing, self-restore, list-membership, cited-by, and currency gating — the exact mission failure the audit exists to prevent. **The 18-row sign-off list in [[onchain-completeness]] §3 is the tickable detail; this is its one-line home on the ceremony sheet.** Priced by the §B gas-snapshot gate before final sign-off.

10. **KEL/account foundation — FREEZE-SENSITIVE (surfaced and specialist-re-audited 2026-07-11; canonical design [[kel]]).** Decide five items as one architecture boundary: (a) per-principal co-located KEL/admission home selected by sparse Ethereum-L1 locator, with the proof/migration/dead-home costs explicitly funded; (b) build/review the KEL-aware lane in v2 vs knowingly forfeit first-class delegation and historical authorization; (c) default-on append-once legacy upgrade commitment; (d) allow a deployed smart account to call the authority-home registry directly once, while excluding ERC-1271/6492 from canonical inception/portable authority; and (e) personal principals non-transferable. **Recommend [[kel]] §23's defaults.** This decision precedes envelope vectors, the kernel confluence claim, identity/read grades, `act`, P-256/WebAuthn activation, and the final pledge scope.

**James acknowledgments** (accepted positions only James can reopen — not pending ratifications): the old year-0 exclusion of smart-contract-only wallet users and FM-2 `metaHash` deferral are reopened by A10; [[kel]] supplies a portable-key setup path and replaces challenge-window-only recovery with a prior-commitment design.

## C. One-final-freeze pledge — written scope (closing gap G10)

> **Scope is provisional until A10.** The KEL pass changes what can honestly be called generic admission confluence and adds authority/epoch words, home/evidence lanes, receipts, and a bounded identity ABI.

**Bound by the pledge (Etched):** the envelope wire format + signature/replay domain + all DOMAIN_* constants **including the KEL authority seam**; the record-kind table + reserved-key rows + derivation math (the deterministic-ids v2 delta); kernel admission semantics (including the explicit authoritative-home/evidence boundary), comparator, G-set revocation, auto-intern + frozen record/identity read ABI + storage layout + genesis manifest; the identity-word taxonomy + canonical KEL/control/grant/recovery/receipt/algoTag/WebAuthn formats.
**NOT bound (Durable/iterable):** view contracts, [[read-lens-spec]] resolution algorithms and grade rendering, SDK surfaces, relayer/lens conformance rules, EASExporter, the apps cookbook, all doctrine documents — these version freely.
**Additive-later (reserved slots make retrofit legal):** reviewed new signature suites and PQ algoTags/verifiers within the frozen agility grammar, successor convention activation, witness/checkpoint tooling (never kernel semantics), duplicate-member role, WHITEOUT, salted TAGDEF (`DOMAIN_ANCHOR_SALTED` family) + blinded-TAGDEF disclosure record, REF-to-property annotation role, author-chosen list identityKey mode, datatype-tag extension constants. **KEL machinery, principal/actor binding, and admission receipts are no longer classified as additive-later.**
