## Headline results

- **The pass held.** All nine kickoff questions answered with cause; no mission end breached; no lane's central ruling destroyed. Four fatal-class findings — all real, all repairable pre-ceremony.
- **The CRDT dismissal was right about the kernel and wrong about the read layer.** CRDT-merge as a **read-time fold** over trusted authors' signed op-claims is the same construction as lens resolution; the merged doc is a *view*, not a record (author = recovered signer unbroken); kernel changes: zero. The genuinely novel result: **lens-scoped convergence** — per-reader op-set selection is a collaboration primitive the CRDT literature wants and doesn't have deployed. BUT the red team proved the **public** op-fold case impossible (public + permissionless + convergent + bounded-reader-cost — a four-property impossibility): public docs route to revision-DAG + curation; op-fold docs require blinded capability-gated containers. **Corollary: the privacy tier is availability infrastructure**, validating the privacy pull-forward.
- **"Write permission" is retired** — decomposed into five wants (visibility/authority/read-exclusion/write-exclusion-GONE/retraction); delegated authorship = the read-side `act` row (kernel verifies nothing, ever); macaroons rejected (fails verify-don't-trust), UCAN attenuation grammar adopted without its bearer carrier.
- **The time model survived pressure with one reversal:** admittedAt is existence-since evidence ONLY — the "freshness via admittedAt" idea (verify-time-model fix 6, bound by three docs) is fail-open even at home; the checkpoint recency beacon replaces it. claimedAt converged to always-present-uint64/performed-at-only/forward-only-falsifier. The comparator fence (fix 4) was silently dropped by its own lane and is restored.
- **Privacy landed:** salted-TAGDEF family fully pinned (HKDF salts legal — P9 roaming confirmed), encryptionKey row minted with a separate KEM registry, keyWrap ruled TAG-only after the red team found the dual-role PIN overturned a stated deterministic-ids exclusion AND its occurrence key was a public recipient-set oracle; crypto-shred = the only hard delete; WHITEOUT re-encoded as `/.well-known/whiteout` + REF-PIN.
- **Four fatals (all repaired in the synthesis):** F1 public-op-fold impossibility (pattern demoted); F2 fix-6 freshness fail-open (direction split adopted); F3 keyWrap occurrence-key oracle (random defaults); F4 keyWrap dual-role vs stated exclusion (TAG-only, override path named). Elevated to ceremony-blocking: the path-grammar contradiction (two lanes disagreed; a red team mis-reported agreement — one pin, MAX_NAME_BYTES=255), the movedTo evaluator vectors, the R3 event re-cut (drafted set falsified log-only-sync).
- **Process finding worth keeping:** the digest layer silently dropped a red team's two fatals — **the file set, not the digest set, is the record**; the critic caught it by diffing files against digests.
